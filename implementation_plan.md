# ポケモン風ゲーム + State Parser 統合計画

ユーザーの「ファイル通りにして（Hexダンプ等の解析）」「どうやってゲーム始めるの？」「スタートボタンがない」という全ての要望を満たすため、解析ツールとゲームを統合したWebアプリケーションを作成します。

## 提案される変更

1. **カートリッジ（Stateファイル）ロード機能**
   - まず `rom (1).state` をロードする画面（ファイル入力）を用意します。
2. **スクリプト通りの解析（ファイル通りに）**
   - ロード後、Pythonスクリプトと全く同じようにタイトル、ゲームコード、HEXダンプを画面に表示します。
3. **ゲームスタート機能（スタートボタン）**
   - 解析が完了すると、明確な「START GAME」ボタンが出現します。
4. **ポケモンバトルの開始（ゲームとして）**
   - STARTボタンを押すと、以前作成した「リザードン vs カメックス」のターン制バトル画面へシームレスに移行し、ゲームが遊べるようになります。

### [MODIFY] pokemon.html
- 初期画面：ファイルロードと解析結果表示、および「START」ボタン。
- ゲーム画面：Canvasを使用したポケモンバトルUI（初期状態は非表示）。

### [MODIFY] pokemon.css
- サイバー/解析ツール風のスタイルと、ゲームボーイ風バトル画面のスタイルを両立させる。
- 「START」ボタンを点滅させて目立たせる。

### [MODIFY] pokemon.js
- Stateファイルのロードと解析ロジック（Hexダンプ等）。
- STARTボタンクリックによる画面切り替え。
- ポケモンバトルのロジック（以前の完成版をベースに復元）。

## 検証プラン
- `index.html` から遷移後、ファイルをロードし、STARTボタンを押せるか。
- START後にポケモンバトルが正常に機能するか。
